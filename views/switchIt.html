Description

Inspired by fun and fast phone games like Candy Crush, MonsterBusters, or Bejeweled that people love to play for a fun distraction and a simple brain stretch!

All game pieces are present and visible at the beginning of the game. The player must reorganize the pieces so they match along an entire row or column to get points!

Reorganization is achieved in one way: Switch your key piece (top left piece in its own box) with any piece on the board.

Above is the first wireframe and planning I did for the switchIt game. Although crude, this sketch helped me organize my thoughts and strategy before writing code.

The Approach

WHAT

I went for a clean look and feel UI-wise, and focused on making a game that can progress and grow with the user.

Some cool features are:

  • Dynamically created gameArray and HTML board
  • Multiple Levels with different size boards
  • Ability to add levels and monsters easily in an object array and have the game include them automatically.

WHY

From the beginning, my goal was to create a game with a dynamic game board that was able to reform depending on the level requirements.

HOW

I avoided hardcoding variables and elements and set the game up to be easily scalable by pointing all my code to objects and object arrays that store information about each level.
