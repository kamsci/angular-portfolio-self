Description

This app is a fun take on a common industry process: inventory and purchasing. Instead of creating the standard shopping cart experience, I experimented with a slightly different concept: borrowing and lending inventory among friends.

The idea is you and your friends all own various toys, tools, gadgets, and media that you would be willing to lend to each other. This app is a place for you to keep inventory of all those items you would lend to a friend or could borrow from a friend. Each user can see items they can borrow separate from items they lend, or items they already have borrowed. I purposely did not include contact info in the first iteration for privacy reasons and because, as friends, you would already have their contact info.

In order to make the process easier to showcase in a portfolio, I decided not to include user authentication. If I were to release this product to actually be used, I would create a second app for production and add user authentication and profiles.

The Approach

WHAT

The goal was to separate frontend action and activity from the back end. Angular is all front end, talking to Node and Express on the backend, which in turn talks to a SQL/Postgresql database.

Some cool features are:

  • Ability to add items attached to a specific user and the user can edit and delete their items
  • User can see items they can borrow separate from items they lend, or items they already have borrowed.
  • A user can select a friend to lend their item to and the friend's name will appear as the 'borrower' with the current date.

WHY

When thinking about coding from a DRY, RESTfull, and MVC perspective, you want to have clear roles for each tool and not allow bleeding over. Express doesn't do any frontend routing, and Angular doesn't talk to the database. This project was an exercise in making the very front (HTML) talk to the very backend (DB) through the proper use of appropriate channels.

HOW

Introducing... the PEAN stack. Postgresql - Express - Angular - Node.js with Gulp on the side for easy developer expansion.

Views are set up in Angular for page navigation and display of content. Angualr talks to Express.js through $http calls. Express routes are purely to take request from angular and retrieve and calculate data from the backend. All express routes start in the root folder followed by "/api/" to identify them as non-front end views.
