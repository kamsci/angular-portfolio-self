Collecting data about your process and progress toward your goals is great, but without clear and thoughtful data visualization it's difficult to actually use. I picked a hot topic in healthcare today, patients that are being readmitted the hospital within 30 days of their release. There are 5 diagnosis that are being monitored for improvement and these 5 are represented in my data.

In a past career life, I worked with healthcare data to create reports for hospitals, home health agencies, long-term care facilities, and hospices to help them understand their data. Healthcare facilites are accountable to the government to improve quality and performance in order to retain full funding. You can imagine then that understanding their data and using their data to create and reach performance goals was vital.

In one project I had to use excel charts and a PDF printer to create meaningfull data charts for our clients. I loved createing the project and I always wondered what I could have build with dev resources. This project tests my future capabilities for dynamic visual reporting.

The Approach

WHAT

By law, hospital performance data is made publically available. I used two APIs from data.medicare. gov. The first to get the general data for all the hospitals and the second second to get individual the measure performance data for reporting.

Some cool features are:

  • Javascript data calculation that creates benchmark data by state and by nation.
  • Ablility to add multiple datasets to use in the report and altenate between.
  • Interactive and easy to read Highcharts.

WHY

This project was approached with the Model-View-Controller and RESTful routes methods in mind. The goal was to use a database and web server to create an interactive application. Although I am not sure sure I would place the data route under the dashboard route a second time, it definately helped me undsertand more about the importance of RESTful routing.

HOW

Express.js, a module from Node.js, what what I used to set up the web service. For the apis, I used the Request module. Passport along with other custome middleware was used for authenticaion and authorization. PostgreSQL served as the database.

The most complicated (and satisfying) part of the project was writing the calculations to turn the raw data into what I needed for the Highcharts on the Dashboard. This was accomplished using Javascript and my knowledge about data analysis.
